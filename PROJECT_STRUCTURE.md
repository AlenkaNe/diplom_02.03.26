# Структура проекта: Автоматические vs Ручные файлы

## 🔴 АВТОМАТИЧЕСКИ СГЕНЕРИРОВАННЫЕ (НЕ ТРОГАТЬ!)

### Backend:

#### `backend/dist/` - ВСЯ ПАПКА
- **Что это:** Скомпилированный JavaScript код из TypeScript
- **Когда создается:** При запуске `npm run build` или `nest build`
- **Содержит:** 
  - `.js` файлы (скомпилированный код)
  - `.d.ts` файлы (типы TypeScript)
  - `.js.map` файлы (source maps для отладки)
- **Важно:** Эту папку можно удалить - она пересоздастся при сборке

#### `backend/node_modules/` - ВСЯ ПАПКА
- **Что это:** Установленные npm-пакеты
- **Когда создается:** При запуске `npm install`
- **Важно:** Никогда не редактировать вручную! Можно удалить и переустановить через `npm install`

#### `backend/package-lock.json`
- **Что это:** Автоматически создается npm при установке пакетов
- **Когда создается:** При `npm install`
- **Важно:** Не редактировать вручную, но коммитить в git нужно

### Frontend:

#### `frontend/node_modules/` - ВСЯ ПАПКА
- **Что это:** Установленные npm-пакеты
- **Когда создается:** При запуске `npm install`

#### `frontend/package-lock.json`
- **Что это:** Автоматически создается npm при установке пакетов
- **Когда создается:** При `npm install`

## 🟡 КОНФИГУРАЦИОННЫЕ (Частично автоматические, частично ручные)

### Backend:

#### `backend/package.json`
- **Что это:** Описание проекта и зависимостей
- **Статус:** Создан автоматически при `nest new`, но можно редактировать
- **Что редактировать:** 
  - ✅ `dependencies` и `devDependencies` (добавление пакетов)
  - ✅ `scripts` (команды запуска)
  - ❌ Остальное лучше не трогать

#### `backend/tsconfig.json`
- **Что это:** Настройки компилятора TypeScript
- **Статус:** Создан автоматически при `nest new`, но можно настраивать
- **Когда менять:** Если нужно изменить настройки компиляции

#### `backend/tsconfig.build.json`
- **Что это:** Дополнительные настройки для сборки
- **Статус:** Создан автоматически при `nest new`

#### `backend/nest-cli.json`
- **Что это:** Настройки NestJS CLI
- **Статус:** Создан автоматически при `nest new`
- **Когда менять:** Редко, только для особых настроек

#### `backend/eslint.config.mjs`
- **Что это:** Настройки линтера (проверка кода)
- **Статус:** Создан автоматически при `nest new`, можно настраивать

#### `backend/test/jest-e2e.json`
- **Что это:** Настройки тестов
- **Статус:** Создан автоматически при `nest new`

### Frontend:

#### `frontend/package.json`
- **Что это:** Описание проекта и зависимостей
- **Статус:** Создан вручную, но можно редактировать
- **Что редактировать:** 
  - ✅ `dependencies` и `devDependencies`
  - ✅ `scripts`

## 📋 КРАТКАЯ СВОДКА

### ❌ НЕ ТРОГАТЬ (автоматические):
- `node_modules/` (обе папки)
- `dist/` (backend)
- `package-lock.json` (оба файла)

### ⚙️ МОЖНО НАСТРАИВАТЬ (конфигурация):
- `package.json` (оба) - добавлять пакеты
- `tsconfig.json` - настройки TypeScript
- `nest-cli.json` - настройки NestJS

### ✏️ ВАША РАБОТА (логика):
- `backend/src/` - весь исходный код бэкенда
- `frontend/server.js` - сервер фронтенда
- `frontend/public/` - HTML, CSS, JS фронтенда

## 🎯 ГДЕ ПИСАТЬ КОД?

### Backend:
- **Новые API маршруты:** `backend/src/food/food.controller.ts` или создайте новый контроллер
- **Бизнес-логика:** `backend/src/food/food.service.ts` или создайте новый сервис
- **Модели данных:** `backend/src/food/entities/` или создайте новые entity

### Frontend:
- **HTML:** `frontend/public/index.html`
- **Стили:** `frontend/public/styles.css`
- **JavaScript:** `frontend/public/script.js`
- **Настройки сервера:** `frontend/server.js`

---

## 💡 СОВЕТЫ

1. **Всегда работайте в папке `src/` (backend) и `public/` (frontend)**
2. **Не редактируйте файлы в `dist/` - они перезапишутся при сборке**
3. **Не трогайте `node_modules/` - используйте `npm install` для установки пакетов**
4. **При добавлении новых функций создавайте новые файлы в `src/`**
