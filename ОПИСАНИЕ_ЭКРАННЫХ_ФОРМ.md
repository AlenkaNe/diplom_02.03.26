# Описание экранных форм веб-приложения "Здоровое питание"

## Общая структура приложения

Веб-приложение состоит из 6 основных экранов, доступных через навигационное меню:
1. Про здоровое питание (информационная страница)
2. ПП для похудения (информационная страница)
3. Калькулятор калорий
4. Поиск продуктов
5. Дневник питания
6. Профиль

---

## 1. ЭКРАН "ПРОФИЛЬ" (profile.html)

### 1.1. Форма входа (авторизация)
**Назначение:** Авторизация существующего пользователя в системе

**Поля формы:**
- **Логин** (обязательное поле, текстовое)
- **Пароль** (обязательное поле, скрытое)

**Элементы управления:**
- Кнопка "Войти" (отправка формы)

**Особенности:**
- Форма отображается в режиме вкладок (переключение между "Вход" и "Регистрация")
- При успешной авторизации автоматически загружается информация о пользователе
- Отображаются сообщения об ошибках (красный фон) или успехе (зеленый фон)

---

### 1.2. Форма регистрации
**Назначение:** Создание нового аккаунта пользователя

**Поля формы:**
- **Логин** (обязательное поле, текстовое)
- **Пароль** (обязательное поле, скрытое)
- **Имя** (необязательное поле, текстовое)
- **Фамилия** (необязательное поле, текстовое)
- **Пол** (необязательное поле, выпадающий список):
  - Не указан
  - Мужской
  - Женский
- **Возраст** (необязательное поле, числовое, диапазон 1-120)
- **Рост (см)** (необязательное поле, числовое, диапазон 50-250)
- **Вес (кг)** (необязательное поле, числовое с десятичными, диапазон 20-500, пример: 68.5)

**Элементы управления:**
- Кнопка "Зарегистрироваться" (отправка формы)

**Особенности:**
- Поля "Имя" и "Фамилия" расположены в одной строке (двухколоночная сетка)
- Поля "Пол" и "Возраст" расположены в одной строке
- При успешной регистрации автоматически выполняется вход и загрузка профиля

---

### 1.3. Отображение информации о пользователе
**Назначение:** Просмотр данных профиля авторизованного пользователя

**Отображаемая информация (в виде карточек):**
- Логин
- Имя
- Фамилия
- Пол
- Возраст
- Рост (см)
- Вес (кг)
- Цель (из калькулятора)
- Уровень активности
- Суточная норма калорий (ккал)
- Индекс массы тела (ИМТ)

**Элементы управления:**
- Кнопка "Редактировать данные" (показывает/скрывает форму редактирования)
- Кнопка "Выйти" (выход из системы)

**Особенности:**
- Информация отображается в виде сетки карточек
- Если данные не указаны, отображается "Не указано" или "Не рассчитано"

---

### 1.4. Форма редактирования профиля
**Назначение:** Изменение данных профиля пользователя

**Поля формы:**
- **Логин** (обязательное поле, текстовое)
- **Имя** (необязательное поле, текстовое)
- **Фамилия** (необязательное поле, текстовое)
- **Пол** (необязательное поле, выпадающий список):
  - Не указан
  - Мужской
  - Женский
- **Возраст** (необязательное поле, числовое, диапазон 1-120)
- **Рост (см)** (необязательное поле, числовое, диапазон 50-250)
- **Вес (кг)** (необязательное поле, числовое с десятичными, диапазон 20-500)

**Элементы управления:**
- Кнопка "Сохранить изменения" (отправка формы)

**Особенности:**
- Форма скрыта по умолчанию, появляется при нажатии "Редактировать данные"
- Поля предзаполнены текущими значениями из профиля
- После сохранения форма скрывается и данные обновляются

## 2. ЭКРАН "КАЛЬКУЛЯТОР КАЛОРИЙ" (calorie-calculator.html)

### 2.1. Форма расчета калорий
**Назначение:** Расчет суточной нормы калорий и макронутриентов

**Поля формы:**
- **Пол** (обязательное поле, выпадающий список):
  - Выберите пол
  - Мужчина
  - Женщина
- **Возраст (лет)** (обязательное поле, числовое, диапазон 1-120)
- **Рост (см)** (обязательное поле, числовое, диапазон 50-250)
- **Вес (кг)** (обязательное поле, числовое с десятичными, диапазон 20-300)
- **Уровень активности** (обязательное поле, выпадающий список):
  - Выберите уровень
  - Минимальная (сидячий образ жизни) - коэффициент 1.2
  - Низкая (легкая активность 1-3 раза в неделю) - коэффициент 1.375
  - Средняя (умеренная активность 3-5 раз в неделю) - коэффициент 1.55
  - Высокая (высокая активность 6-7 раз в неделю) - коэффициент 1.725
  - Очень высокая (ежедневные интенсивные тренировки) - коэффициент 1.9
- **Цель** (обязательное поле, выпадающий список):
  - Выберите цель
  - Похудение
  - Поддержание веса
  - Набор массы
- **Формула расчета** (обязательное поле, выпадающий список):
  - Выберите формулу
  - Харриса-Бенедикта
  - Миффлина-Сан Жеора

**Элементы управления:**
- Кнопка "Рассчитать" (отправка формы и выполнение расчетов)

**Особенности:**
- Все поля обязательны для заполнения
- После расчета результаты отображаются на той же странице

---

### 2.2. Блок результатов расчета
**Назначение:** Отображение результатов расчета калорий и макронутриентов

**Отображаемая информация:**

**Индекс массы тела (ИМТ):**
- Числовое значение ИМТ
- Визуальная шкала с указателем (диапазоны: <18.5, 18.5-24.9, 25.0-29.9, 30+)
- Текстовое описание значения ИМТ (недостаточный вес, норма, избыточный вес, ожирение)
- Примечание о том, что ИМТ используется для предварительной оценки

**Суточная норма калорий:**
- Отображается крупным шрифтом в формате: "Суточная норма калорий: [значение] ккал"

**Макронутриенты:**
- Круговая диаграмма (график) с распределением белков, жиров и углеводов
- Список макронутриентов с цветовой индикацией:
  - **Белки** (синий цвет) - значение в ккал
  - **Жиры** (оранжевый цвет) - значение в ккал
  - **Углеводы** (зеленый цвет) - значение в ккал

**Особенности:**
- Результаты сохраняются в профиле пользователя (если пользователь авторизован)
- Данные можно использовать для настройки дневника питания

## 3. ЭКРАН "ПОИСК ПРОДУКТОВ" (index.html)

### 3.1. Форма поиска продуктов
**Назначение:** Поиск продуктов в базе данных по названию

**Поля формы:**
- **Поле ввода** (текстовое, placeholder: "Введите название продукта...")

**Элементы управления:**
- Кнопка "Найти" (запуск поиска)

**Особенности:**
- Поиск выполняется в реальном времени при нажатии кнопки
- Отображается индикатор загрузки во время выполнения запроса

---

### 3.2. Таблица результатов поиска
**Назначение:** Отображение найденных продуктов

**Колонки таблицы:**
- **ID** - идентификатор продукта
- **Название** - наименование продукта
- **Бренд** - производитель/бренд
- **Состав** - описание состава продукта
- **Размер порции** - стандартный размер порции
- **Подробнее** - кнопка для просмотра детальной информации о нутриентах
- **Действия** - кнопка "Добавить в дневник"

**Особенности:**
- Если результатов нет, отображается сообщение "Нет результатов"
- Каждая строка содержит продукт с возможностью просмотра деталей и добавления в дневник

---

### 3.3. Модальное окно "Информация о нутриентах"
**Назначение:** Детальная информация о питательной ценности продукта

**Содержимое:**
- Заголовок: "Информация о нутриентах"
- Подзаголовок: "Расчет на 100г продукта"
- Таблица или список нутриентов:
  - Калории (ккал)
  - Белки (г)
  - Жиры (г)
  - Углеводы (г)
  - Дополнительные микроэлементы (если доступны)

**Элементы управления:**
- Кнопка закрытия (×) в правом верхнем углу

**Особенности:**
- Модальное окно открывается поверх основного контента
- Закрывается при клике на кнопку закрытия или вне области окна

---

### 3.4. Модальное окно "Добавить продукт в дневник питания"
**Назначение:** Добавление найденного продукта в дневник питания

**Поля формы:**
- **Тип приема пищи** (обязательное поле, выпадающий список):
  - Выберите тип приема пищи
  - Завтрак
  - Обед
  - Ужин
  - Перекус
  - (и другие типы из базы данных)
- **Вес (граммы)** (обязательное поле, числовое с десятичными, минимум 1, шаг 0.1, placeholder: "Введите вес в граммах")

**Элементы управления:**
- Кнопка "Добавить в дневник" (отправка формы)
- Кнопка закрытия (×) в правом верхнем углу

**Особенности:**
- Типы приемов пищи загружаются динамически из базы данных
- После успешного добавления отображается сообщение
- Требуется авторизация пользователя

---

## 4. ЭКРАН "ДНЕВНИК ПИТАНИЯ" (diary.html)

### 4.1. Селектор даты
**Назначение:** Выбор даты для просмотра записей дневника

**Элементы управления:**
- Поле выбора даты (date picker)
- По умолчанию устанавливается текущая дата

**Особенности:**
- При изменении даты автоматически загружаются записи за выбранный день

---

### 4.2. Блок итогов за день
**Назначение:** Отображение суммарных значений калорий и макронутриентов за день

**Отображаемые показатели (в виде карточек):**
- **Калории** (красный цвет) - общее количество ккал
- **Белки** (синий цвет) - общее количество в граммах
- **Жиры** (оранжевый цвет) - общее количество в граммах
- **Углеводы** (зеленый цвет) - общее количество в граммах

**Особенности:**
- Блок отображается только если есть записи за выбранный день
- Значения рассчитываются автоматически на основе всех записей

---

### 4.3. Секции приемов пищи
**Назначение:** Группировка и отображение записей по типам приемов пищи

**Структура:**
Каждый тип приема пищи отображается в отдельной секции:
- Заголовок секции с названием типа приема пищи (например, "Завтрак", "Обед")
- Список продуктов, добавленных в этот прием пищи

**Карточка продукта содержит:**
- **Название продукта** (крупным шрифтом)
- **Вес** (в граммах, серым цветом)
- **Нутриенты** (в виде сетки):
  - Калории (ккал) - красный цвет
  - Белки (г) - синий цвет
  - Жиры (г) - оранжевый цвет
  - Углеводы (г) - зеленый цвет

**Особенности:**
- Если для типа приема пищи нет записей, отображается сообщение "Нет записей"
- Все типы приемов пищи отображаются всегда (даже если пустые)
- Записи отсортированы по времени добавления

---

### 4.4. Сообщения об ошибках
**Назначение:** Информирование пользователя о проблемах

**Типы сообщений:**
- Ошибка авторизации (красный фон, белый текст): "Необходима авторизация. Пожалуйста, войдите в систему."
- Ошибка загрузки данных (красный фон, белый текст): "Ошибка загрузки данных: [описание ошибки]"

**Особенности:**
- Сообщения отображаются в верхней части страницы
- Автоматически скрываются при успешной загрузке данных

**Особенности:**

- Текст: "Здоровое питание."

## ТЕХНИЧЕСКИЕ ОСОБЕННОСТИ ФОРМ

### Валидация полей
- Обязательные поля помечены атрибутом `required`
- Числовые поля имеют ограничения по диапазону (min/max)
- Поля с десятичными значениями используют атрибут `step="0.1"`

### Обработка ошибок
- Все формы отображают сообщения об ошибках
- Сообщения имеют цветовую индикацию (красный - ошибка, зеленый - успех)
- Ошибки валидации обрабатываются на клиенте и сервере

### Авторизация
- Некоторые функции требуют авторизации (добавление в дневник, просмотр дневника)
- Авторизация выполняется через JWT токены, хранящиеся в cookies
- При отсутствии авторизации отображаются соответствующие сообщения

### Адаптивность
- Формы используют адаптивную верстку (grid layout)
- Элементы подстраиваются под размер экрана
- Модальные окна адаптируются для мобильных устройств
